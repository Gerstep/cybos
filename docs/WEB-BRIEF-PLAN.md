# Web Brief Feature - Implementation Plan

*Morning brief as a web UI, scheduled at 8am daily*

---

## Overview

**User Story:** At 8am daily, a headless process generates a morning brief, transforms it to a web page, and opens it in my browser. When I wake up and check my Mac, the first thing I see is my personalized daily brief.

**Components:**
1. Enhanced `/cyber-brief` command (adds Synthesis + Strategic Leverage)
2. Markdown → JSON parser
3. Lightweight HTTP server (Hono)
4. React web app (pre-built, served statically)
5. macOS launchd scheduling
6. Orchestrator script

## Phase 1: Brief Enhancements

### 1.1 System Synthesis

Add a synthesis section generated by Claude after all data is gathered.

**Location in brief:** After header, before Priority Actions

**Implementation:**
- After gathering Telegram, Email, Calendar, GTD data
- Spawn a focused agent (Haiku) with all context
- Generate 2-3 sentence synthesis highlighting:
  - Overall day tone (busy, light, critical)
  - Key blockers or urgent items
  - What success looks like today

**Markdown format:**
```markdown
## System Synthesis

The investment pipeline is surging. You have 2 urgent messages from founders
awaiting response—Elena's term sheet feedback is blocking the Sequoia call.
Today's success: clear the blocking items before the 10:30 call.
```

### 1.2 Strategic Leverage

AI-scored high-impact opportunities surfaced from gathered data.

**Location in brief:** After Today's Schedule, before Messages

**Data sources:**
- Goals from `context/who-am-i.md` (Active Priorities section)
- GTD items from `GTD.md` (# Next section)
- Messages/emails waiting for response
- Calendar prep requirements
- User-defined rules (configurable)

**Scoring algorithm:**

### 2.2 Server Implementation

**Tech choice:** Hono (lightweight, TypeScript-native, fast startup)

```typescript
// scripts/brief-server.ts
import { Hono } from 'hono'
import { serve } from '@hono/node-server'
import { serveStatic } from '@hono/node-server/serve-static'
import { cors } from 'hono/cors'
import { parseBrief } from './brief-parser'
import { existsSync, readFileSync } from 'fs'
import { resolve } from 'path'

const PORT = 3847
const BRIEFS_DIR = resolve(__dirname, '../content/briefs')
const STATIC_DIR = resolve(__dirname, '../docs/web-brief/dist')

const app = new Hono()

// CORS for local dev
app.use('/*', cors())

// API: Get brief by day
app.get('/api/brief/:day', async (c) => {
  const day = c.req.param('day') // 'today' | 'yesterday' | 'MMDD-YY'

  try {
    const briefPath = resolveBriefPath(day)

    if (!existsSync(briefPath)) {
      return c.json({
        error: 'Brief not found',
        path: briefPath,
        suggestion: day === 'today' ? 'Run /cyber-brief to generate' : null
      }, 404)
    }

    const markdown = readFileSync(briefPath, 'utf-8')
    const data = parseBrief(markdown)
    return c.json(data)

  } catch (error) {
    return c.json({
      error: 'Parse error',
      message: error.message
    }, 500)
  }
})

// API: List available briefs
app.get('/api/briefs', async (c) => {
  const files = readdirSync(BRIEFS_DIR)
    .filter(f => f.endsWith('.md'))
    .sort()
    .reverse()
    .slice(0, 10)

  return c.json({ briefs: files })
})

// API: Health check
app.get('/api/health', (c) => {
  return c.json({ status: 'ok', port: PORT })
})

// Static files (React app)
app.use('/*', serveStatic({ root: STATIC_DIR }))

// Redirect /brief/today to React app with query param
app.get('/brief/:day', (c) => {
  return c.redirect(`/?day=${c.req.param('day')}`)
})

// Helper: resolve brief path
function resolveBriefPath(day: string): string {
  if (day === 'today') {
    const d = new Date()
    const mm = String(d.getMonth() + 1).padStart(2, '0')
    const dd = String(d.getDate()).padStart(2, '0')
    const yy = String(d.getFullYear()).slice(-2)
    return resolve(BRIEFS_DIR, `${mm}${dd}-${yy}.md`)
  }

  if (day === 'yesterday') {
    const d = new Date()
    d.setDate(d.getDate() - 1)
    const mm = String(d.getMonth() + 1).padStart(2, '0')
    const dd = String(d.getDate()).padStart(2, '0')
    const yy = String(d.getFullYear()).slice(-2)
    return resolve(BRIEFS_DIR, `${mm}${dd}-${yy}.md`)
  }

  // Direct filename: MMDD-YY
  return resolve(BRIEFS_DIR, `${day}.md`)
}

console.log(`Brief server starting on http://localhost:${PORT}`)
serve({ fetch: app.fetch, port: PORT })
```

// docs/web-brief/src/api.ts
const API_BASE = 'http://localhost:3847/api'

export async function fetchBrief(day: 'today' | 'yesterday' | string): Promise<BriefData> {
  const res = await fetch(`${API_BASE}/brief/${day}`)
  if (!res.ok) {
    const error = await res.json()
    throw new Error(error.message || 'Failed to fetch brief')
  }
  return res.json()
}
```

### 3.3 Orchestrator Script

```bash
#!/bin/bash
# scripts/morning-brief.sh
# Orchestrates morning brief generation and browser opening

set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"
LOG_FILE="$PROJECT_DIR/.cybos/logs/morning-brief.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" >> "$LOG_FILE"
}

log "=== Morning Brief Started ==="

# 1. Generate brief via Claude headless
log "Generating brief..."
cd "$PROJECT_DIR"

if claude --headless "/cyber-brief" >> "$LOG_FILE" 2>&1; then
    log "Brief generated successfully"
else
    log "ERROR: Brief generation failed"
    # Continue anyway to show error in UI
fi

# 2. Check if server is running
if ! curl -s http://localhost:3847/api/health > /dev/null 2>&1; then
    log "Server not running, it should be started by launchd"
    # Server should be running via separate launchd daemon
fi

# 3. Wait a moment for any file writes to complete
sleep 2

# 4. Open browser
log "Opening browser..."
open "http://localhost:3847/?day=today"

log "=== Morning Brief Complete ==="
```

## Phase 4: Installation

### 4.1 Setup Script

```bash
#!/bin/bash
# scripts/install-brief.sh
# One-time setup for morning brief feature

set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

bun install

echo "✓ Server daemon installed and running"

echo "Test manually:"

## Phase 5: Update cyber-brief Command

### 5.1 Changes to `.claude/commands/cyber-brief.md`

```markdown
### 2.5 GENERATE SYNTHESIS (after gathering data)

Using all gathered data, generate a synthesis:

**Spawn Haiku agent** with this prompt:
```
You are generating a morning brief synthesis for the user.

Data gathered:
- Telegram: [X] unread conversations
- Emails: [X] important/unread
- Calendar: [X] events today
- GTD: [X] pending tasks

Key items:

Generate a 2-3 sentence synthesis that:

2. Highlights the single most important blocker or opportunity

### 2.6 CALCULATE STRATEGIC LEVERAGE

For each item across all sources, calculate leverage score:

1. Load rules from `config/leverage-rules.yaml`
2. Load goals from `context/who-am-i.md` (Active Priorities section)
3. Score each item against all rules
4. Take top 3-5 items with score >= 7.0
5. For each high-leverage item, generate:
   - Problem statement
   - Impact if not addressed
   - Recommended action

**Output format for each:**
```markdown
### [Title]
- **Score:** [X.X]
- **Problem:** [What needs attention]
- **Impact:** [What happens if ignored]
- **Source:** [Where this came from]
- **Recommended action:** [Specific next step]
```
```
